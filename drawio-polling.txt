# Блок-схема для Draw.io: Алгоритм опроса результата выполнения

## Инструкция для Draw.io:
1. Используйте прямоугольники для процессов
2. Ромбы для условий
3. Овалы для начала/конца
4. Стрелки с подписями "Да"/"Нет"

---

## Схема:

**НАЧАЛО**
↓
**Инициализация**
- submissionId (ID задачи)
- maxAttempts = 30 (макс. попыток)
- intervalMs = 100 (интервал в мс)
- attempt = 0
↓
**Проверка: attempt < maxAttempts?**
├─ НЕТ → **Ошибка: Превышено время ожидания** → **КОНЕЦ**
└─ ДА ↓

**Отправка HTTP запроса**
- GET к Judge0 API
- URL: /submissions/{submissionId}
- Параметры: base64_encoded=true
↓
**Проверка: Запрос успешен?**
├─ НЕТ → **Ошибка: Не удалось получить результат** → **КОНЕЦ**
└─ ДА ↓

**Парсинг JSON ответа**
- Получить result
↓
**Проверка: result.status.id > 2?**
├─ НЕТ (В очереди или обработка) ↓
│   **Ожидание**
│   - Пауза intervalMs миллисекунд
│   ↓
│   **Увеличить счетчик**
│   - attempt = attempt + 1
│   ↓
│   (Возврат к проверке attempt < maxAttempts)
│
└─ ДА (Выполнено) ↓

**Декодирование результатов**
- Декодировать stdout из base64
- Декодировать stderr из base64
- Декодировать compile_output из base64
- Декодировать message из base64
↓
**Формирование результата**
- id = submissionId
- status (статус выполнения)
- stdout (вывод программы)
- stderr (ошибки)
- time (время выполнения)
- memory (использованная память)
- exit_code (код завершения)
↓
**Возврат результата**
↓
**КОНЕЦ**

---

## Пояснения:

**Статусы Judge0:**
- 1 = В очереди
- 2 = Обрабатывается
- 3 = Успешно выполнено
- 4+ = Ошибки (неверный ответ, ошибка компиляции и т.д.)

**Параметры:**
- Максимум попыток: 30
- Интервал между попытками: 100 мс
- Максимальное время ожидания: 3 секунды

**Назначение:**
Алгоритм опрашивает сервер Judge0 для получения результата выполнения кода. Повторяет запросы пока код не выполнится или не истечет время ожидания.
