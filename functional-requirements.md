# Описание функциональных требований проекта

## Общее описание
Веб-приложение для совместной разработки и выполнения кода в реальном времени с поддержкой задач, проектов и коллаборации.

## Роли пользователей

### 1. Роль «Гость» (Неавторизованный пользователь)
Функции пользователя с ролью «Гость»:
- Регистрация нового аккаунта в системе
- Авторизация через существующий аккаунт
- Просмотр публичной информации о платформе
- Просмотр списка доступных задач для программирования
- Фильтрация задач по языкам программирования
- Поиск задач по названию и описанию
- Просмотр описания задач без возможности выполнения

### 2. Роль «Пользователь» (Авторизованный пользователь)
Функции пользователя с ролью «Пользователь»:
- Создание новых проектов
- Просмотр списка своих проектов
- Редактирование названия и настроек проектов
- Удаление собственных проектов
- Создание комнат для совместной работы внутри проектов
- Управление доступом к комнатам (только владелец / любой по ссылке)
- Приглашение других пользователей в комнаты через ссылку
- Совместное редактирование кода в реальном времени (collaborative editing)
- Просмотр курсоров и действий других участников комнаты
- Создание и управление файлами в проекте (создание, редактирование, удаление)
- Выполнение кода на различных языках программирования (JavaScript, Python, C++, Java и др.)
- Просмотр результатов выполнения кода (stdout, stderr, время, память)
- Видео и аудио связь с участниками комнаты через WebRTC
- Текстовый чат с участниками комнаты
- Решение задач по программированию
- Создание сессий для решения задач
- Управление доступом к сессиям задач (владелец / любой по ссылке)
- Отслеживание прогресса выполнения задач
- Просмотр истории попыток решения задач
- Просмотр статистики активности (календарь активности)
- Просмотр истории посещенных комнат
- Удаление комнат из истории

### 3. Роль «Администратор» (Системный администратор)
Функции пользователя с ролью «Администратор»:
- Все функции роли «Пользователь»
- Просмотр всех проектов в системе
- Удаление любых проектов и комнат
- Просмотр статистики использования системы
- Управление пользователями (блокировка, удаление)
- Создание, редактирование и удаление задач для программирования
- Настройка языков программирования и их конфигураций
- Управление лимитами выполнения кода (rate limiting)
- Просмотр логов активности пользователей
- Мониторинг нагрузки на сервер выполнения кода (Judge0)
- Управление настройками WebRTC сервера
- Настройка параметров коллаборативного редактирования (Hocuspocus)

## Модуль аналитики и статистики

Приложение содержит модуль анализа с отображением следующей статистики:

### Для пользователей:
- Календарь активности с отображением количества решенных задач по дням
- Общее количество решенных задач
- Количество попыток решения задач
- Распределение решенных задач по языкам программирования
- История выполнения кода с результатами (успех/ошибка)
- Время, затраченное на решение задач
- Прогресс выполнения задач (начато/завершено)

### Для администраторов:
- Общее количество пользователей в системе
- Количество активных пользователей за период
- Общее количество проектов и комнат
- Статистика использования языков программирования
- Количество выполненных запросов к Judge0 API
- Статистика по задачам (популярность, процент решения)
- Логи активности пользователей (STARTED, COMPLETED, SUBMITTED)
- Нагрузка на сервер коллаборации (количество активных соединений)
- Статистика использования WebRTC (количество видео-сессий)

## Дополнительные функциональные требования

### Система контроля доступа:
- Двухуровневая система доступа к комнатам: OWNER (только владелец) и ANYONE_WITH_LINK (любой по ссылке)
- Автоматическое отслеживание истории доступа к комнатам
- Аутентификация через Supabase Auth
- Проверка прав доступа при подключении к WebSocket (Hocuspocus)

### Ограничения и безопасность:
- Rate limiting для выполнения кода: 10 запросов в минуту на пользователя
- Таймаут выполнения кода: максимум 3 секунды (30 попыток × 100мс)
- Изоляция данных между комнатами
- Персистентность состояния документов в PostgreSQL
- Эфемерные сессии для задач (без сохранения в БД)

### Технические особенности:
- Collaborative editing на основе Yjs и Hocuspocus
- WebRTC для видео/аудио связи между участниками
- Интеграция с Judge0 API для выполнения кода
- Поддержка множества языков программирования
- Real-time синхронизация курсоров и изменений
- Автоматическое сохранение состояния документов
