# Блок-схема для Draw.io: Алгоритм ограничения частоты запросов

## Инструкция для Draw.io:
1. Используйте прямоугольники для процессов
2. Ромбы для условий
3. Овалы для начала/конца
4. Стрелки с подписями "Да"/"Нет"

---

## Схема:

**НАЧАЛО**
↓
**Вход**
- userId (ID пользователя)
↓
**Получить текущее время**
- now = текущее время в миллисекундах
↓
**Поиск записи пользователя**
- entry = найти запись для userId в Map
↓
**Проверка: Запись существует И now <= entry.resetAt?**
├─ НЕТ (Новое окно или окно истекло) ↓
│   **Создание нового временного окна**
│   - count = 1 (первый запрос)
│   - resetAt = now + 60000 (сброс через 1 минуту)
│   ↓
│   **Сохранить запись**
│   - Сохранить entry для userId в Map
│   ↓
│   **Возврат результата**
│   - allowed = true (разрешено)
│   - remaining = 9 (осталось запросов)
│   - resetAt (время сброса)
│   ↓
│   **КОНЕЦ**
│
└─ ДА (Окно активно) ↓

**Проверка: entry.count >= 10?**
├─ ДА (Лимит достигнут) ↓
│   **Возврат отказа**
│   - allowed = false (запрещено)
│   - remaining = 0 (запросов не осталось)
│   - resetAt = entry.resetAt (когда сбросится)
│   ↓
│   **КОНЕЦ**
│
└─ НЕТ (Лимит не достигнут) ↓

**Увеличить счетчик**
- entry.count = entry.count + 1
↓
**Возврат результата**
- allowed = true (разрешено)
- remaining = 10 - entry.count (осталось запросов)
- resetAt = entry.resetAt (время сброса)
↓
**КОНЕЦ**

---

## Пояснения:

**Константы:**
- Временное окно: 60000 мс (1 минута)
- Максимум запросов: 10 запросов в минуту

**Структура данных:**
```
entry = {
  count: число запросов в текущем окне
  resetAt: время сброса счетчика (в миллисекундах)
}
```

**Принцип работы:**
1. Для каждого пользователя хранится счетчик запросов
2. Счетчик сбрасывается каждую минуту
3. Если пользователь превысил 10 запросов - отказ
4. Если окно истекло - создается новое окно

**Назначение:**
Защита API от перегрузки. Ограничивает количество запросов на выполнение кода до 10 в минуту для каждого пользователя.
